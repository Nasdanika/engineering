    • User Guide
    • Engineering - One of definitions of engineering is �The action of working artfully to bring something about.� Bringing something of value about is usually quite a lot of effort.
      • Flow - This package contains classes for defining �how to do things� in a non-linear visual form using journeys.
        • Activity - Activity is an element of a Journey.
        • Call - Call is a specialization of transition which has blocking �request-reply� semantics where the calling activity waits for a response from callee.
        • Choice - Choice is a conditional pseudo-state.
        • End - End is a final pseudo-state with only inputs and no outputs indicating completion of a journey.
        • EntryPoint - Entry point is a pseudo-state displayed as a circle on the border of the containing partition journey through which transitions enter the journey.
        • ExitPoint - Exit point is a pseudo-state displayed as a crossed circle on the border of the containing partition journey through which transitions leave the journey.
        • ExpansionInput - Expansion input is a pseudo-state which can be used instead of partition start pseudo-state to indicate that the partition journey input is a collection and the journey is executed for each element in the collection.
        • ExpansionOutput - Expansion output is a pseudo-state which can be used instead of partition end pseudo-state to indicate that the partition journey output is a collection and the output transitions are executed for each element in the collection.
        • Fork - Fork is a pseudo-state which splits journey elements execution into parallel branches.
        • InputPin - Input is a pseudo-state displayed as a square on the border of the containing parition journey through which calls initiate the journey.
        • Join - Join is a pseudo-state which merges journey parallel execution branches.
        • Journey - Journey is a composite activity.
        • JourneyElement - Journey element is a source and target of transitions and calls.
        • OutputPin - Output pin is a pseudo-state displayed as a square on the border of the containing partition journey through which calls are made to other journey elements.
        • PseudoState - A pseudo-state is a base class for transient journey elements in the journey graph.
        • Service - Service is a reference to a shared activity used in multiple journeys.
        • Start - Start is an initial pseudo-state with only outputs and no inputs indicating start of a journey.
        • Transition - Transition connects two journey elements, e.g. activities and is navigated when execution of the source element is completed and execution of the target element shall begin.
      • Representation - Model element representations.
        • ComponentDiagram - Component diagram visualizing dependencies and containment.
      • Adaptable
      • Aim - Aim is the base class for things to which engineered capabilities and other aims can be aligned.
      • Alignable - Something that can be aligned to aims by alignments - aims or endeavors.
      • Alignment - An association class between aims and engineered capabilities.
      • Allocation - Allocation of engineer�s capacity to spend on a particular engineered element and issue category on a particular endeavor.
      • Appearance - Appearance allows to customize HTML UI generation process.
      • AppearanceEntry - An EMap entry class to UI action name to model element appearance mapping.
      • Capability - Capability is something that can be required by an issue to start working on it.
      • Capacity - Capacity is contained under Engineer and defines how much total effort or funds the containing engineer has to spend on a particular endeavor.
      • Decision - Decision is an issue, which is also an aim, i.e. endeavors, such as issues, and other aims such as other decisions may be aligned to it.
      • Directory - Directory is a container of named elements, e.g. documents, links, or other directories.
      • Document - Document may contain content, table of contents, and other documents as sections.
      • Duration - Duration is an amount of time in the ISO-8601 Duration format.
      • Endeavor - Endeavor is a base class for things which take time, effort, and money to achieve some benefit - increments, and engineered capabilities - issues, features, and releases.
      • Engineer - Engineers are entities, people or organizations, owning engineered elements and working on endeavors, e.g. issues.
      • EngineeredCapability - Engineered capability extends both endeavor and capability and is a base class for issue, feature, and release.
      • EngineeredElement - Engineered element is something that is �brought about� by engineers working on issues associated with the engineered element.
      • Event - Event is a named temporal.
      • Feature - Feature is a unit of consumption of product functionality.
      • Forum - Discussion forum which can contain sub-forums and discussion topics.
      • Goal - Goal is an aim defined under a persona and its sub-types - engineer and organization.
      • Increment - Increment is a period of time to plan releases and schedule issues for.
      • Instant - Instant is a single instantaneous point on the time-line.
      • Issue - Issue is a unit of work to accomplish to achieve a desired outcome on its containing engineered element, e.g.: New functionality in a module, New or refined persona goals, New skills acquired by an engineer, Improved organization processes.
      • IssueCategory - A mechanism for grouping/tagging issues.
      • IssuePriority - Issue priority, e.g. Low, Medium, High.
      • IssueSeverity - Issue severity, e.g. Feature, Minor, Major, Crash.
      • IssueStatus - Issue status, e.g. Open, In Progress, Done.
      • KeyResult - Key Results are specific measures used to track achievement of their containing objective.
      • Link - Link to an external URL.
      • MemberAppearance - Member appearance defines how a structural feature or an operation is shown in the UI.
      • MemberAppearanceEntry - An EMap entry class to support feature or operation name to member appearance mapping.
      • Message - A message in a discussion topic authored by an engineer on a specific date.
      • ModelElement - Model element is the root class for all other model classes.
      • ModelElementAppearance - Model element appearance defines how a model element is shown in the UI.
      • ModelElementAppearanceEntry - An EMap entry class to support model element class name to model element appearance mapping.
      • Module - Module is a part of product or another module.
      • NamedElement - Named element, as the name suggests, is a base class for model elements with a name.
      • NamedElementReference - References a named element.
      • Note - Note is used to track issue progress as well as to discuss issue implementation.
      • Objective - Objective is something that should be achieved as part of endeavor such as an increment or a release.
      • Organization - Organization is an engineer which can contain other engineers, and organizations as such.
      • PackageAppearance - Package appearance contains maps of: Package class names in kebab case to model element appearance Sub-package names in kebab case to package apperance.
      • PackageAppearanceEntry - An EMap entry class to support sub-package name to package appearance mapping.
      • Period - Amount of time between two time points.
      • Persona - Personas represent represent the different types of consumers of products and services.
      • Principle - Principles are general rules and guidelines, intended to be enduring and seldom amended, that inform and support the way in which an engineer or an organization sets about achieving its goals or addressing goals of their consumers modeled as personas...
      • Product - Product is an object or a system made available for consumer use.
      • Release - Release is a unit of delivering product features to its consumers.
      • SectionStyle - Defines how sections are rendered.
      • TableOfContents - Table of Contents configuration - role, depth, header.
      • Temporal - Temporal represents a point on the time-line - absolute (instant) or relative to another temporal (base and offset).
      • Topic - Topic is a group of related messages in a forum.
